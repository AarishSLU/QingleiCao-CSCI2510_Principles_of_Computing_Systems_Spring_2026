3. What happened when SIGINT was sent with kill?
The sleeper process terminated immediately. The printing stopped suddenly with no other message.

4. Another signal tested:
I used SIGTERM, which is signal 15.
When I sent it to the sleeper process, terminal 1 stopped printing and said "Terminated".

5. What happened when call_sleeper was run and interrupted with CTRL-C?
When I ran call_sleeper and pressed CTRL C, both the sleeper and the caller terminated. The caller did not print its success message.

6. Signal handling code:

void ignore_sigint(int signum) {
    printf("\nIgnoring SIGINT in caller.\n");
}

signal(SIGINT, ignore_sigint);

8. Another situation where signal handling might be useful:
Signal handling could be useful when a server needs to reload its configuration without fully shutting down. A signal could notify the program to reload settings or reopen log files while it keeps running.
